一、总体思路
操作系统是计算机系统的核心软件，其涉及的进程管理、处理器调度、进程同步、
死锁避免、内存管理、磁盘调度等算法是理解系统工作原理的关键。本题目要求开发一
个可视化操作系统核心算法展示系统，通过图形界面或 Web 页面，动态模拟并展示各
类经典 OS 算法的执行过程。
二、项目技术方案
模块 1：处理器调度模拟模块
功能要求：
• 支持以下调度算法：
o 先来先服务（FCFS）
o 时间片轮转（RR，可设置时间片）
o 短进程优先（SJF，抢占）
o 最高响应比优先（HRN）
• 输入：进程 ID、到达时间、服务时间（可动态添加）
• 输出：
o 完成时间、周转时间、带权周转时间
o 调度顺序图（甘特图形式）
可视化建议：
• 用时间轴或甘特图展示调度顺序。
模块 2：生产者-消费者同步模拟模块
功能要求：
• 模拟多个生产者线程和多个消费者线程，消费者需要连续取 n 个产品后，其他
消费者进程才可以取产品。
• 使用信号量机制实现同步与互斥。
• 支持以下控制：
o 设置缓冲区大小
o 设置生产/消费速度（可调速）
o 实时显示缓冲区状态（满/空/中间状态）
o 连续取产品个数 n
可视化建议：
• 用图形化格子表示缓冲区，颜色区分空/满/占用中。
模块 3：银行家算法演示模块
功能要求：
• 实现银行家算法，判断系统是否处于安全状态。
• 支持动态输入：
o 资源总数（A, B, C）
o 各进程的最大需求、已分配资源
o 进程请求资源（m, n, p）
• 输出：
o 安全序列
o 是否可分配资源
o 若分配后是否仍安全
可视化建议：
• 表格形式展示进程资源状态
• 安全序列用箭头或步骤条展示
模块 4：页面置换算法模拟模块
功能要求：
• 支持以下页面置换算法：
o OPT（最优置换）
o LRU（最近最久未使用）
o FIFO（先进先出）
• 输入：
o 页面引用序列（可手动输入或随机生成）
o 内存物理块数
• 输出：
o 每次页面置换的页面号
o 缺页次数、缺页率
o 内存状态变化过程
可视化建议：
• 表格或矩阵方式展示内存块随时间的变化
• 高亮缺页事件
模块 5：磁盘移臂调度模拟模块
功能要求：
• 支持以下调度算法：
o SSTF（最短寻道时间优先）
o SCAN（电梯算法，可设置方向）
• 输入：
o 磁道请求序列
o 初始磁头位置
• 输出：
o 响应顺序
o 磁头移动总量（寻道总长度）
o 磁头移动轨迹图
可视化建议：
• 折线图或坐标图展示磁头移动路径
